# Phase 1 Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Wire Supabase email/password auth to Prisma profiles, client auth state, header UI, and server-side sim rate limits.

**Architecture:** Server actions handle sign-up/sign-in/sign-out and profile creation; Zustand + AuthProvider sync client state; middleware protects routes.

**Tech Stack:** Next.js 16, Supabase SSR, Prisma, Zustand, shadcn/ui

## Global Constraints

- Self-hosted Supabase at `supabase.local`
- Email/password only (OAuth deferred)
- `Profile.id` = Supabase `auth.users.id`
- Server-side sim counter on `Profile`
